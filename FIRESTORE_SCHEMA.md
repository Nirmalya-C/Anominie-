# Firestore Data Models

This document describes the Firestore collections and document schemas used by Anominie.

---

## Collection: `users`

Stores a profile document for every authenticated session.

**Document ID:** Firebase Authentication `uid` (anonymous or linked)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `username` | `string` | ✅ | Display name chosen by the user (max 30 chars) |
| `country` | `string` | ✅ | Country selected during profile setup |
| `createdAt` | `timestamp` | ✅ | Server timestamp – when the profile was first created |
| `updatedAt` | `timestamp` | ✅ | Server timestamp – last time the profile was modified |

**Example document (`users/{uid}`):**
```json
{
  "username": "Jordan",
  "country": "USA",
  "createdAt": "2024-01-15T10:30:00Z",
  "updatedAt": "2024-01-15T10:30:00Z"
}
```

---

## Collection: `messages`

Stores all chat messages in the global room. New documents are added with
`db.collection('messages').add(...)`, so Firestore auto-generates the ID.

**Document ID:** Auto-generated by Firestore

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `uid` | `string` | ✅ | Firebase Auth uid of the sender |
| `username` | `string` | ✅ | Display name of the sender at the time of posting |
| `country` | `string` | ✅ | Country of the sender at the time of posting |
| `text` | `string` | ✅ | Message body (max 500 chars, enforced client-side) |
| `timestamp` | `timestamp` | ✅ | Server timestamp – when the message was stored |

**Example document (`messages/{autoId}`):**
```json
{
  "uid": "abc123xyz",
  "username": "Jordan",
  "country": "USA",
  "text": "Hello everyone!",
  "timestamp": "2024-01-15T10:35:00Z"
}
```

## Firestore Security Rules

Add the following rules in the Firebase console under
**Firestore Database → Rules** to protect your data:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only read/write their own profile
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }

    // Any authenticated user (including anonymous) can read and create messages
    match /messages/{messageId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null
                    && request.resource.data.uid == request.auth.uid
                    && request.resource.data.text is string
                    && request.resource.data.text.size() <= 500;
      // No updates or deletes allowed
      allow update, delete: if false;
    }
  }
}
```

## Indexes

For the real-time message listener (`orderBy('timestamp', 'asc').limitToLast(50)`)
Firestore creates the required single-field index automatically.

No composite indexes are required for the current queries.
